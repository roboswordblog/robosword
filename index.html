How I learned Godot in under 48 hours for a game jam

It all started after I tried to learn Unity in within 2 days. Then I tried to find a game jam. I saw one on itch.io called "Godot Wild Game Jam". I decided that I would join it. I joined 48 hours before the game jam. I made it my mission to finish learning it within 48 hours. To achieve this goal, I started with a Brackey's platformer video. I stopped in the middle of the series because while saving one of the files, I accedently deleted it. The skills that I gained from it includes how to use a tilemap, player movements, physics, the basics of gdscripts, and a little more. That was the only video which I was able to fully watch before the game jam. After that video, I still had 24 hours left. So, I tried making my own game. Using this I felt a lot more comfortable using godot, but my gdscipt skills were still kinda of bad. So the night before, I studied from the GDScript Api

func _ready() -> null: is when the scene is started

func _physics_process(delta) -> null: is when you need to update something physically, usually by using move_and_slide()

My expierience with Unity

I tried out unity over this week, and I want to share my expierence. I wanted to try to speedrun unity and learn it within 3 days. I started with a unity 3d video, which helped me get through the basics, and I learned some things in Unity3d. Then I tried taking a unity2d tutorial, but I quickly relized that even though unity is usable with 2d applications, it is not that easy. It is clear that Unity is mostly a 3d game engine. 2d mode is also really hard to use for a beginner. I would recommend people to use unity for 3d games, but for 2d, you should use a different game engine, or maybe even use pygame.

My Honest opinion on Java Swing

So far, I have made 2 Projects using Java swing. A text editor and a library app. Both of them are in my github and on my projects page. Before this, I mainly used python. I would give this a 4/5. Swing was really easy for me to use as a beginner to Java. The problem with Swing howerver, is that it was very hard for me to learn to use Layouts. I did not understand it at all, and I just chose to avoid it completely. In the situations which I had to use the layouts, I ended up having to use google to search everything up. It also does not come with rounded corners built into the buttons, but thats not the most important thing. I would say for a beginner, this would be a really good tool to get started with. If you are new to java, I highly recomment making a project in Swing Java, because it can also help you learn some of the fundamentals and/or file handling in java.

Django vs Flask

Django and flask are two different libraries made for web development backend. Django comes with everything in it, but is much much more complecated to set up. Flask is really easy to set up, but it doesn't come with everything set up. If you are doing a miniproject or a hackathon, you should choose flask. If you are going to release something into the real world, you should do django.

Another thing to consider is routing. In flask, you can route using this

@app.route("/action", methods=["Get", "Post"])

In django, you have to create a file specifically for routing, and link it to your view files. This makes it inefficient for small projects and hackathons, because it requires too much time. Setting up is also pretty hard in django, and setting up static files is really easy in flask unlike in django
