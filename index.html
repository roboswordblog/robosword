My Honest opinion on Java Swing

So far, I have made 2 Projects using Java swing. A text editor and a library app. Both of them are in my github and on my projects page. Before this, I mainly used python. I would give this a 4/5. Swing was really easy for me to use as a beginner to Java. The problem with Swing howerver, is that it was very hard for me to learn to use Layouts. I did not understand it at all, and I just chose to avoid it completely. In the situations which I had to use the layouts, I ended up having to use google to search everything up. It also does not come with rounded corners built into the buttons, but thats not the most important thing. I would say for a beginner, this would be a really good tool to get started with. If you are new to java, I highly recomment making a project in Swing Java, because it can also help you learn some of the fundamentals and/or file handling in java.

Django vs Flask

Django and flask are two different libraries made for web development backend. Django comes with everything in it, but is much much more complecated to set up. Flask is really easy to set up, but it doesn't come with everything set up. If you are doing a miniproject or a hackathon, you should choose flask. If you are going to release something into the real world, you should do django.

Another thing to consider is routing. In flask, you can route using this

@app.route("/action", methods=["Get", "Post"])

In django, you have to create a file specifically for routing, and link it to your view files. This makes it inefficient for small projects and hackathons, because it requires too much time. Setting up is also pretty hard in django, and setting up static files is really easy in flask unlike in django
